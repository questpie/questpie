# Config API

QUESTPIE uses `runtimeConfig()` to define application runtime settings and a file convention with codegen to compose the full app.

## Runtime Config

```ts
// questpie.config.ts
import { runtimeConfig, pgBossAdapter, SmtpAdapter } from "questpie";

export default runtimeConfig({
  app: { url: process.env.APP_URL! },
  db: { url: process.env.DATABASE_URL! },
  secret: process.env.BETTER_AUTH_SECRET!,
  queue: { adapter: pgBossAdapter({ connectionString: process.env.DATABASE_URL! }) },
  email: {
    adapter: new SmtpAdapter({
      transport: { host: "localhost", port: 1025, secure: false },
    }),
  },
  cli: {
    migrations: { directory: "./src/migrations" },
    seeds: { directory: "./src/seeds" },
  },
});
```

## Config Options

| Field | Required | Description |
|---|---|---|
| `app.url` | Yes | Public URL of the application |
| `db.url` | Yes | PostgreSQL connection string |
| `secret` | Yes | Secret key for auth tokens |
| `storage` | No | Storage config (driver, basePath) |
| `email` | No | Email adapter (default: console) |
| `queue` | No | Queue adapter (default: pg-boss) |
| `search` | No | Search adapter instance |
| `kv` | No | KV adapter (default: in-memory) |
| `logger` | No | Logger config (level, pretty) |
| `realtime` | No | Realtime adapter |
| `defaultAccess` | No | Default access rules for all entities |
| `translations` | No | Translation messages |
| `cli` | No | CLI tool configuration |

## Entity Factories

Entities are defined using factories and placed in convention directories:

### `collection(name)` — from `"#questpie"`

Create a collection builder. Place in `collections/` directory.

```ts
// collections/posts.ts
import { collection } from "#questpie";

export const posts = collection("posts")
  .fields(({ f }) => ({
    title: f.text({ required: true }),
    status: f.select({
      options: [
        { value: "draft", label: { en: "Draft" } },
        { value: "published", label: { en: "Published" } },
      ],
      default: "draft",
      required: true,
    }),
  }));
```

### `global(name)` — from `"#questpie"`

Create a global builder. Place in `globals/` directory.

```ts
// globals/site-settings.ts
import { global } from "#questpie";

export const siteSettings = global("siteSettings")
  .fields(({ f }) => ({
    siteName: f.text({ required: true }),
    logo: f.upload({ to: "assets" }),
  }));
```

### `fn(definition)` — from `"questpie"`

Define a server function. Place in `functions/` directory.

```ts
// functions/get-available-slots.ts
import { fn } from "questpie";
import { z } from "zod";

export default fn({
  schema: z.object({ barberId: z.string(), date: z.string() }),
  handler: async ({ input, collections }) => {
    return collections.appointments.find({
      where: { barber: input.barberId },
    });
  },
});
```

### `job(definition)` — from `"questpie"`

Define a background job. Place in `jobs/` directory.

```ts
// jobs/send-confirmation.ts
import { job } from "questpie";
import { z } from "zod";

export default job({
  name: "send-confirmation",
  schema: z.object({ appointmentId: z.string() }),
  handler: async ({ payload, collections, email }) => {
    const appointment = await collections.appointments.findOne({
      where: { id: payload.appointmentId },
    });
    if (!appointment) return;

    await email.sendTemplate({
      template: "appointmentConfirmation",
      to: "customer@example.com",
      input: { appointment },
    });
  },
});
```

### `email(definition)` — from `"questpie"`

Define an email template. Place in `emails/` directory.

```ts
// emails/appointment-confirmation.ts
import { email } from "questpie";
import { z } from "zod";

export default email({
  name: "appointmentConfirmation",
  schema: z.object({ appointment: z.any() }),
  handler: ({ input }) => ({
    subject: `Appointment Confirmed`,
    html: `<h1>Your appointment is confirmed!</h1>`,
  }),
});
```

## Modules

Modules are static objects registered in `modules.ts`:

```ts
// modules.ts
import { adminModule, auditModule } from "@questpie/admin/server";

export default [adminModule, auditModule] as const;
```

| Module | Package | What it adds |
|--------|---------|-------------|
| `starterModule` | `questpie` | User collection, assets, auth, default access |
| `adminModule` | `@questpie/admin/server` | Admin UI server functions, field types, preferences |
| `auditModule` | `@questpie/admin/server` | Audit logging, cleanup job, dashboard widget |

## Codegen

Run `questpie generate` to produce `.generated/index.ts` with:

- All entity imports and registration
- `AppContext` module augmentation for typed handler contexts
- `AppConfig` type for client SDK
- `createContext()` for typed context in scripts
