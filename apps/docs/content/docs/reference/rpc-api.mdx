# Functions API

Functions provide type-safe server procedures callable from the client SDK. They are defined with `fn()` from `"questpie"`, validated with Zod schemas, and support both JSON and raw HTTP modes.

## Defining Functions

### `fn(definition)` — JSON Function

Validated input/output with automatic Zod schema enforcement.

```ts
import { fn } from "questpie";
import { z } from "zod";

export default fn({
  schema: z.object({
    barberId: z.string(),
    date: z.string(),
    serviceId: z.string().optional(),
  }),
  handler: async ({ input, app, session }) => {
    const barber = await app.api.collections.barbers.findOne({
      where: { id: input.barberId },
    });
    // ... compute available slots
    return { slots: availableSlots };
  },
});
```

**Definition fields:**

| Field | Type | Required | Description |
|---|---|---|---|
| `schema` | `ZodType` | Yes | Zod schema for input validation |
| `handler` | `(ctx) => Promise<TOutput>` | Yes | Function implementation |
| `access` | `(ctx) => boolean \| Promise<boolean>` | No | Access control |
| `outputSchema` | `ZodType` | No | Zod schema for output validation |

### `fn(definition)` — Raw Function

Direct access to `Request` and `Response` for streaming, file downloads, or custom protocols.

```ts
import { fn } from "questpie";
import { z } from "zod";

export default fn({
  schema: z.object({ format: z.enum(["csv", "json"]) }),
  raw: true,
  handler: async ({ input, app }) => {
    const csv = await generateCsv(app);
    return new Response(csv, {
      headers: { "Content-Type": "text/csv" },
    });
  },
});
```

### Handler Context

Both function types receive a context object:

```ts
{
  input: TInput;           // Validated input (JSON functions)
  request: Request;        // Raw HTTP request (raw functions)
  app: Questpie;           // app instance
  session: Session | null; // Authenticated user
  locale: string;          // Current locale
}
```

For strongly typed `app`, `db`, and `session`, cast inside the handler:

```ts
import { getContext } from "questpie";
import type { App } from '../app';

handler: async (ctx) => {
  const { app, db, session } = getContext<App>(ctx);
  // app, db, session are now strongly typed
}
```

## File-Based Routing

Functions live in a `functions/` directory. Nested folders create nested routes automatically — no manual router composition needed.

```
functions/
  booking/
    getAvailableSlots.ts   → POST /api/functions/booking/getAvailableSlots
    createBooking.ts       → POST /api/functions/booking/createBooking
    cancelBooking.ts       → POST /api/functions/booking/cancelBooking
  reports/
    generateMonthly.ts     → POST /api/functions/reports/generateMonthly
    exportCsv.ts           → POST /api/functions/reports/exportCsv
```

Each file exports a default `fn()` definition. The file path determines the RPC route — no `router()` calls required.

## Mounting Functions

Functions are discovered automatically from the `functions/` directory. No `rpc` option is needed in the fetch handler config:

```ts
import { createFetchHandler } from "questpie";

const handler = createFetchHandler(app, {
  basePath: "/api",
});
```

## Access Control

Functions support an `access` callback:

```ts
fn({
  schema: z.object({ id: z.string() }),
  access: ({ session }) => {
    if (!session) return false;
    return session.user.role === "admin";
  },
  handler: async ({ input, app }) => {
    // Only admins reach here
  },
})
```

If `access` returns `false`, the function responds with `403 FORBIDDEN`.

## Client Invocation

### Via Client SDK

```ts
import { createClient } from "@questpie/tanstack-query";

const client = createClient<typeof app>({ baseUrl: "/api" });

// Type-safe call
const slots = await client.functions.booking.getAvailableSlots({
  barberId: "b1",
  date: "2024-03-15",
});
```

### Via HTTP

```
POST /api/functions/booking/getAvailableSlots
Content-Type: application/json

{ "barberId": "b1", "date": "2024-03-15" }
```

Response:

```json
{ "slots": [...] }
```

## Error Handling

- Invalid input → `400 BAD_REQUEST` with Zod field errors
- Access denied → `403 FORBIDDEN`
- Handler throws → `500 INTERNAL_SERVER_ERROR` (original error in dev mode)
- Function not found → `404 NOT_FOUND`
