This document is the authoritative wire reference for what flows between the admin and the preview iframe today. The TypeScript source of truth lives in `packages/admin/src/client/preview/types.ts` вҖ” every shape on this page mirrors a type exported from there.

For the high-level user story (when to enable, how the workspace consumes it, per-field tuning), see [Visual Edit Workspace](./visual-edit). This page is for implementors building custom iframe code or extending the protocol.

## Origin Validation

Every postMessage is sent with an **explicit target origin** вҖ” never `"*"` вҖ” and validated on receipt against the resolved peer origin.

**Admin side** (`PreviewPane`):

- Resolves the iframe's origin once from the configured preview URL via `new URL(url, window.location.href).origin`.
- Drops outbound messages and warns in dev when the origin can't be resolved.
- Wraps `iframe.contentWindow.postMessage(message, targetOrigin)` in a `try`/`catch` so a non-serializable payload (`DataCloneError`) is logged in dev and swallowed in production.

**Iframe side** (`useCollectionPreview`):

- Resolves the admin origin once from `document.referrer`.
- Drops inbound messages whose `event.origin` doesn't match.
- When `document.referrer` is empty (e.g. opened directly), accepts only same-origin parents вҖ” never silently trusts an unknown frame.

There is no `sessionId` on the wire; the admin tab's lifetime IS the session.

## Message Envelope

Each message is a **plain object literal** with a `type` discriminant вҖ” there is no shared base shape. Concrete payloads are listed below; `type` is exhaustively covered by `AdminToPreviewMessage` and `PreviewToAdminMessage` in `types.ts`.

The validators `isAdminToPreviewMessage` / `isPreviewToAdminMessage` accept any object whose `type` matches a known string. Future additions only need to register their string in the validator's set and the union type вҖ” there's no version negotiation handshake.

## Sequence Numbers

Only `PATCH_BATCH` carries a sequence number (`seq`). Every other message is unsequenced.

- The admin's patcher (`useFormToPreviewPatcher`) maintains a monotonic counter, incremented per emitted batch. The counter resets to `0` whenever the patcher's `baseline` reference changes (post-COMMIT refetch, locale switch, вҖҰ).
- The iframe (`useCollectionPreview`) keeps a `lastSeqRef`; batches with `seq <= lastSeqRef` are dropped. `INIT_SNAPSHOT`, `COMMIT`, and `FULL_RESYNC` reset `lastSeqRef` to `-1`, so the very next batch is always accepted.

## Admin вҶ’ Iframe

### `PREVIEW_REFRESH`

V1 fallback that asks the iframe to re-run its loader.

```ts
type PreviewRefreshMessage = {
  type: "PREVIEW_REFRESH";
  changedField?: string; // dev-only hint
};
```

The iframe `await`s the consumer's `onRefresh` callback then replies with `REFRESH_COMPLETE`.

### `INIT_SNAPSHOT`

Seeds the iframe's local draft with a full record.

```ts
type InitSnapshotMessage = {
  type: "INIT_SNAPSHOT";
  snapshot: Record<string, unknown>;
  schemaVersion?: string;
  locale?: string;
  stage?: string;
};
```

- `PreviewPane` buffers the most recent payload and replays it on every `PREVIEW_READY` it receives, so this can be sent before the iframe is ready or before the iframe reloads.
- The Visual Edit bridge sends two flavours: the canonical `controller.transformedItem` whenever the loaded record changes, and the **current `react-hook-form` values** on every `PREVIEW_READY` (so unsaved edits survive an iframe reload).

### `PATCH_BATCH`

Mutates the iframe's local draft.

```ts
type PreviewPatchOp =
  | { op: "set"; path: string; value: unknown }
  | { op: "remove"; path: string };

type PatchBatchMessage = {
  type: "PATCH_BATCH";
  seq: number;
  ops: PreviewPatchOp[];
};
```

- `path` is dot-notation (`"title"`, `"meta.seo.title"`, `"items.2.label"`, `"content._values.abc.title"`); numeric segments index arrays.
- Generated by `diffSnapshot(before, after)` вҖ” recursive over plain objects, atomic over arrays. A change to `meta.seo.title` emits a single `set meta.seo.title = "X"` op rather than replacing the whole `meta` subtree.
- The iframe applies via `applyPatchBatch` and acks with `PATCH_APPLIED`.

### `COMMIT`

Marks the local draft as saved.

```ts
type CommitMessage = {
  type: "COMMIT";
  timestamp: number;
  snapshot?: Record<string, unknown>;
};
```

- When `snapshot` is provided, the iframe replaces its draft with it (use this when the server may have changed derived data вҖ” slug, computed fields, relation joins).
- When `snapshot` is omitted, the iframe drops the local draft and re-runs `onRefresh`.

### `FULL_RESYNC`

Discards the iframe's local draft and triggers a re-fetch.

```ts
type FullResyncMessage = {
  type: "FULL_RESYNC";
  reason?: "revert" | "locale-switch" | "stage-switch" | "desync" | "manual";
};
```

The bridge fires this with the right `reason` for delete (`manual`), restore + revertVersion (`revert`), workflow transition (`stage-switch`).

### `FOCUS_FIELD`

Highlights a field in the iframe.

```ts
type FocusFieldMessage = {
  type: "FOCUS_FIELD";
  fieldPath: string;
};
```

The iframe sets `focusedField` and scrolls the matching `data-preview-field="вҖҰ"` element into view.

### `SELECT_BLOCK`

Sets the iframe's selected-block state.

```ts
type SelectBlockMessage = {
  type: "SELECT_BLOCK";
  blockId: string;
};
```

### `SELECT_TARGET`

Newer, kind-aware selection forwarding from the workspace.

```ts
type SelectTargetMessage = {
  type: "SELECT_TARGET";
  fieldPath: string | null;
  kind?: string;
  blockId?: string;
};
```

The iframe maps this to `focusedField` (and updates `selectedBlockId` when `blockId` is set). Unifies the `FOCUS_FIELD` + `SELECT_BLOCK` use case for the Visual Edit Workspace.

### `NAVIGATE_PREVIEW`

Asks the iframe to navigate to a URL.

```ts
type NavigatePreviewMessage = {
  type: "NAVIGATE_PREVIEW";
  url: string;
};
```

- Same-origin URLs only вҖ” the iframe rejects cross-origin URLs and warns in dev.
- Uses `window.location.replace` (not `href`) so the iframe never accumulates a back-button stack.

## Iframe вҶ’ Admin

### `PREVIEW_READY`

Sent by `useCollectionPreview` whenever its message-listener effect runs (initial mount and after a hard reload).

```ts
type PreviewReadyMessage = { type: "PREVIEW_READY" };
```

`PreviewPane` reacts by replaying the buffered `INIT_SNAPSHOT` and firing the consumer's `onReady` callback. The Visual Edit host uses `onReady` to re-seed the iframe with current form values.

### `REFRESH_COMPLETE`

Acks `PREVIEW_REFRESH` after `onRefresh` resolves.

```ts
type RefreshCompleteMessage = {
  type: "REFRESH_COMPLETE";
  timestamp: number;
};
```

### `FIELD_CLICKED`

Sent when a `<PreviewField>` is clicked.

```ts
type FieldClickedMessage = {
  type: "FIELD_CLICKED";
  fieldPath: string;
  blockId?: string;
  fieldType?: "regular" | "block" | "relation";
};
```

The workspace routes this through `mapPreviewClickToSelection` to derive a `VisualEditSelection`.

### `BLOCK_CLICKED`

```ts
type BlockClickedMessage = {
  type: "BLOCK_CLICKED";
  blockId: string;
};
```

### `PATCH_APPLIED`

Optional ack for a `PATCH_BATCH`.

```ts
type PatchAppliedMessage = {
  type: "PATCH_APPLIED";
  seq: number;
  applied: number;
  timestamp: number;
};
```

The admin doesn't currently subscribe to this; it's exposed for telemetry / dropped-batch detection in third-party tooling.

### `RESYNC_REQUEST`

Reserved for the iframe to ask for a fresh `INIT_SNAPSHOT`.

```ts
type ResyncRequestMessage = {
  type: "RESYNC_REQUEST";
  reason?: "stale-seq" | "missing-snapshot" | "manual";
};
```

`useCollectionPreview` doesn't currently fire this вҖ” the patcher handles missing-snapshot edge cases by falling back to `initialDataRef` when applying patches without a draft. Reserved for future protocol extensions.


## Reliability Properties

The protocol is designed for **idempotent convergence** rather than strict request-response semantics:

- **Iframe reloads** never lose state. `PREVIEW_READY` triggers a buffered-snapshot replay AND a form-values re-seed; the next `PATCH_BATCH` applies on top of the freshest state. Convergence is guaranteed because every `set` op is idempotent.
- **Stale batches** are dropped via `seq` comparison вҖ” a delayed `PATCH_BATCH(seq=3)` arriving after `seq=5` is ignored.
- **Race between flush and unmount** is benign: the patcher checks `previewRef.current` at flush time and bails if null. The form-values bridge re-fires on the next ready event, so no data is permanently lost.
- **Non-serializable payloads** never crash the workspace: postMessage is wrapped in `try`/`catch` on both sides, with dev-mode `console.error` and production-mode silent-swallow. The admin's inspector error boundary catches misbehaving field renders.
