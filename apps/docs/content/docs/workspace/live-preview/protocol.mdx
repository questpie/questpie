---
title: Preview Protocol Reference
description: postMessage protocol for the existing FormView + LivePreviewMode iframe preview system.
---

The preview protocol connects the admin form and the frontend iframe. Messages are implementation details of the current Live Preview system: the form is authoritative, the iframe mirrors data, and persistence still goes through the existing save/autosave path.

## Message Families

| Direction        | Message              | Purpose                                            |
| ---------------- | -------------------- | -------------------------------------------------- |
| preview -> admin | `PREVIEW_READY`      | iframe loaded and can receive messages             |
| admin -> preview | `INIT_SNAPSHOT`      | seed iframe local draft from the form snapshot     |
| admin -> preview | `PATCH_BATCH`        | apply unsaved field changes                        |
| preview -> admin | `PATCH_APPLIED`      | acknowledge an applied patch batch                 |
| admin -> preview | `PREVIEW_REFRESH`    | ask iframe to re-run its loader                    |
| preview -> admin | `REFRESH_COMPLETE`   | refresh finished                                   |
| preview -> admin | `FIELD_CLICKED`      | user clicked an annotated frontend field           |
| preview -> admin | `BLOCK_CLICKED`      | user clicked an annotated frontend block           |
| admin -> preview | `FOCUS_FIELD`        | highlight a frontend field                         |
| admin -> preview | `SELECT_BLOCK`       | highlight a frontend block                         |
| preview -> admin | `FIELD_VALUE_EDITED` | user committed an inline scalar edit in the iframe |
| admin -> preview | `COMMIT`             | save succeeded, replace mirror with saved snapshot |
| admin -> preview | `FULL_RESYNC`        | discard local mirror and reload from the loader    |
| preview -> admin | `RESYNC_REQUEST`     | iframe detected mismatch and asks for refresh      |

## Handshake

```mermaid
sequenceDiagram
  participant Admin as Admin FormView
  participant Pane as PreviewPane
  participant Frame as Frontend iframe
  participant Hook as useCollectionPreview

  Admin->>Pane: Open existing Preview button
  Pane->>Frame: Load preview URL with token
  Frame->>Hook: Mount page
  Hook->>Pane: PREVIEW_READY
  Pane->>Frame: INIT_SNAPSHOT(current form data)
  Frame->>Hook: Store draft mirror
  Hook->>Pane: PATCH_APPLIED(seq)
```

## Patch Batch

```mermaid
sequenceDiagram
  participant Form as React Hook Form
  participant Pane as PreviewPane
  participant Frame as Frontend iframe
  participant Hook as useCollectionPreview

  Form->>Pane: watch() observes changed values
  Pane->>Pane: diff previous/current snapshot
  Pane->>Frame: PATCH_BATCH(seq, ops)
  Frame->>Hook: applyPatchBatchImmutable
  Hook->>Hook: Re-render from mirrored data
  Hook->>Pane: PATCH_APPLIED(seq)
```

Patch paths use dot notation:

```ts
type PreviewPatchOp = {
	op: "set" | "remove";
	path: string;
	value?: unknown;
};

type PatchBatchMessage = {
	type: "PATCH_BATCH";
	seq: number;
	ops: PreviewPatchOp[];
	snapshotVersion?: number;
};
```

Examples:

```txt
title
seo.description
content._values.block_123.heading
```

## Inline Edit

```mermaid
sequenceDiagram
  participant User
  participant Field as PreviewField
  participant Pane as PreviewPane
  participant Form as React Hook Form
  participant Frame as Frontend iframe

  User->>Field: Edit annotated text
  Field->>Pane: FIELD_VALUE_EDITED(path, value, inputKind)
  Pane->>Pane: Validate origin, session, path, kind, and value
  Pane->>Form: setValue(path, value, dirty + validate + touch)
  Form->>Pane: watch() emits updated snapshot
  Pane->>Frame: PATCH_BATCH(seq, ops)
  Frame->>Field: Render mirrored value
```

Suggested payload:

```ts
type FieldValueEditedMessage = {
	type: "FIELD_VALUE_EDITED";
	path: string;
	value: unknown;
	inputKind: "text" | "textarea" | "number" | "boolean";
	blockId?: string;
	fieldType?: "regular" | "block" | "relation";
};
```

The admin must only call `form.setValue` after the message is validated against the active schema and supported editable field types.

## Commit

```mermaid
sequenceDiagram
  participant User
  participant Form as FormView
  participant DB as Database
  participant Pane as PreviewPane
  participant Frame as Frontend iframe

  User->>Form: Save, autosave, or Cmd+S
  Form->>DB: Existing create/update mutation
  DB-->>Form: Saved record snapshot
  Form->>Pane: Save succeeded
  Pane->>Frame: COMMIT(saved snapshot)
  Frame->>Frame: Replace local mirror baseline
```

`COMMIT` does not save from the iframe. It tells the iframe that the existing form save succeeded and that the saved snapshot is now the baseline.

## Resync

```mermaid
sequenceDiagram
  participant Frame as Frontend iframe
  participant Pane as PreviewPane
  participant Loader as Frontend loader
  participant Form as FormView

  Frame->>Pane: RESYNC_REQUEST(reason)
  Pane->>Frame: FULL_RESYNC(reason)
  Frame->>Loader: Optional route invalidation
  Loader-->>Frame: Fresh loader data
  Pane->>Form: Keep existing form state authoritative
```

Use `FULL_RESYNC` for locale changes, workflow stage changes, restore/revert actions, sequence gaps, schema mismatches, and block tree changes that cannot be safely patched.

## Security Rules

```mermaid
flowchart TD
  Message["Incoming iframe message"] --> Origin{"Origin allowed?"}
  Origin -- "no" --> Drop["Drop"]
  Origin -- "yes" --> Session{"Active preview session?"}
  Session -- "no" --> Drop
  Session -- "yes" --> Type{"Known message type?"}
  Type -- "no" --> Drop
  Type -- "yes" --> Shape{"Payload shape valid?"}
  Shape -- "no" --> Drop
  Shape -- "yes" --> Path{"Editable schema path?"}
  Path -- "no" --> Drop
  Path -- "yes" --> Size{"Serializable and bounded?"}
  Size -- "no" --> Drop
  Size -- "yes" --> Apply["Apply to form or preview mirror"]
```

Admin-side handlers must validate:

- message origin
- active preview session
- message type and payload shape
- field path exists in the active collection schema or block value map
- field type supports the requested inline edit kind
- value is serializable and reasonably sized

Do not use wildcard `postMessage` targets when the preview origin can be resolved from `.preview({ url })`. Do not allow iframe messages to set arbitrary form paths.
