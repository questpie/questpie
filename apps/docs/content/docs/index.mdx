# QUESTPIE Documentation

<Callout type="warn">
**Active Development** — QUESTPIE is a bootstrapped, community-driven framework under active development. The API may still change between releases, but we follow semantic versioning to communicate breaking changes clearly. We do not recommend it for mission-critical production workloads yet. Full API stability is targeted for **v3**.
</Callout>

**One schema. Every output.** QUESTPIE is a file-convention TypeScript backend framework. Define collections, functions, jobs, and hooks as plain files — codegen wires them into a type-safe runtime. No builder chains, no manual registration.

`@questpie/admin` is an optional module that reads your server schema via introspection and generates an admin panel. You can add it when you need an internal interface, or skip it and ship API-only services.

## Runtime First

```text
Required
questpie            — backend runtime (collections, fields, hooks, access, functions)
@questpie/<adapter> — HTTP adapter (Hono, Elysia, or Next.js)

Optional
@questpie/admin     — generated admin interface via introspection
@questpie/tanstack-query, @questpie/openapi, other client tooling
```

The file convention comes first. Drop a collection file, run codegen, and get a fully typed runtime. Admin, SDKs, and OpenAPI all consume the same server definitions instead of redefining schema in multiple places.

## Field Definitions

Fields are defined in collection files using the `f` field builder. Each field maps to database behavior, API validation, and query typing. If admin is installed, those same fields also drive form rendering.

```ts
// src/questpie/server/collections/posts.ts
import { collection } from "#questpie";

export default collection("posts").fields(({ f }) => ({
  title: f.text({ required: true, maxLength: 255 }),
  body: f.richText({ localized: true }),
  cover: f.upload({ to: "assets", mimeTypes: ["image/*"] }),
  author: f.relation({ to: "users" }),
  status: f.select({ options: ["draft", "published"] }),
  publishedAt: f.date(),
  tags: f.array({ of: f.text() }),
  metadata: f.object({
    fields: () => ({
      seoTitle: f.text(),
      seoDescription: f.textarea(),
    }),
  }),
}));
```

## Architecture at a Glance

```text
src/questpie/server/
  ├── collections/         — one file per collection (schema + hooks + access)
  ├── globals/             — one file per global
  ├── functions/           — typed server functions (replaces old RPC)
  ├── jobs/                — background tasks
  ├── modules.ts           — module imports (admin, audit, etc.)
  ├── sidebar.ts           — admin sidebar convention
  ├── dashboard.ts         — admin dashboard convention
  ├── branding.ts          — admin branding convention
  ├── questpie.config.ts   — runtime config (DB, secrets, adapters)
  └── .generated/          — codegen output (never edit manually)

Optional Interface Layer (@questpie/admin)
  └── client module        — field/view/component registries
```
