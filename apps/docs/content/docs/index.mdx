# QUESTPIE CMS

**Headless, extensible CMS framework for TypeScript**

QUESTPIE CMS provides a type-safe backend with a beautiful, config-driven admin interface. Build content management systems without writing boilerplate.

## Key Features

- **Type-Safe Builder Pattern** - Full TypeScript inference from schema to API
- **Config-Driven Admin** - Beautiful UI generated from configuration
- **22+ Field Types** - Text, relations, uploads, rich text, blocks, and more
- **Page Builder** - Visual content editing with custom blocks
- **Framework Agnostic** - Works with Hono, Elysia, Next.js
- **i18n Ready** - Built-in localization for content and UI

## Quick Start

```bash
bun add questpie @questpie/admin @questpie/hono
```

```typescript
// Server — Define your data + admin config
import { q, text, boolean } from "questpie";
import { adminModule } from "@questpie/admin/server";

const cms = q({ name: "my-cms" })
  .use(adminModule)
  .collections({
    posts: q.collection("posts")
      .fields({
        title: text("title").notNull(),
        content: text("content"),
        published: boolean("published").default(false),
      })
      .admin(({ c }) => ({
        label: "Posts",
        icon: c.icon("ph:article"),
      }))
      .list(({ v, f }) => v.table({
        columns: [f.title, f.published],
      }))
      .form(({ v, f }) => v.form({
        fields: [f.title, f.content, f.published],
      })),
  })
  .build({ db: { url: process.env.DATABASE_URL } });

export type AppCMS = typeof cms;
```

```typescript
// Client — Minimal renderer setup
import { qa, adminModule } from "@questpie/admin/client";
import type { AppCMS } from "./server/cms";

const admin = qa<AppCMS>()
  .use(adminModule)
  .branding({ name: "My CMS" });
```

## Architecture

QUESTPIE uses a **server-first** architecture. The server defines WHAT (collections, admin config, dashboard, sidebar). The client defines HOW (renderers, branding):

```
┌─────────────────────────────────────────────────────────────┐
│  Server — q() builder                                        │
│  Defines: collections, fields, hooks, access, validation     │
│  Defines: .admin(), .list(), .form(), .dashboard(), .sidebar │
├─────────────────────────────────────────────────────────────┤
│  Client — qa() builder                                       │
│  Registers: field renderers, view renderers, widget renderers│
│  Configures: branding, locale, custom components             │
├─────────────────────────────────────────────────────────────┤
│  Database (PostgreSQL)                                        │
└─────────────────────────────────────────────────────────────┘
```

**Server (`questpie`)**: Collections, validation, hooks, access control, auth, admin configuration (dashboard, sidebar, list/form views). Single source of truth.

**Client (`@questpie/admin`)**: Renders the admin UI from server config. Handles branding, locale, custom field/view/widget components.

Types flow from server to client, ensuring end-to-end type safety.
