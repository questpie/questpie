# Icon System

QUESTPIE Admin uses `@iconify/react` with the **Phosphor** icon set. Icons are created as serializable references on the server using `c.icon()` and rendered automatically on the client.

## Server-Side: `c.icon()`

In server-side configuration (`.admin()`, `.dashboard()`, `.sidebar()`, `.actions()`), use the `c.icon()` helper:

```typescript
// Server-side: creates a serializable ComponentReference
c.icon("ph:users")         // → { type: "icon", props: { name: "ph:users" } }
c.icon("ph:article-fill")  // Phosphor "fill" variant
c.icon("ph:gear-bold")     // Phosphor "bold" variant
```

### Usage in Config

```typescript
// Collection admin metadata
.admin(({ c }) => ({
  label: { en: "Posts" },
  icon: c.icon("ph:article"),
}))

// Dashboard widgets
.dashboard(({ d, c }) => d.dashboard({
  items: [
    d.stats({
      collection: "posts",
      label: { en: "Total Posts" },
      icon: c.icon("ph:article"),
    }),
  ],
}))

// Sidebar navigation
.sidebar(({ s, c }) => s.sidebar({
  sections: [
    s.section({
      id: "content",
      title: { en: "Content" },
      icon: c.icon("ph:files"),
      items: [
        { type: "collection", collection: "posts", icon: c.icon("ph:article") },
      ],
    }),
  ],
}))

// Custom actions
.actions(({ a, c }) => ({
  custom: [
    a.action({
      id: "publish",
      label: { en: "Publish" },
      icon: c.icon("ph:arrow-up"),
      handler: async ({ itemId }) => { ... },
    }),
  ],
}))
```

## Icon Naming Convention

Icons follow the pattern `ph:icon-name[-variant]`:

| Variant | Suffix | Example |
|---------|--------|---------|
| Regular | (none) | `ph:users` |
| Fill | `-fill` | `ph:users-fill` |
| Bold | `-bold` | `ph:users-bold` |
| Duotone | `-duotone` | `ph:users-duotone` |
| Light | `-light` | `ph:users-light` |
| Thin | `-thin` | `ph:users-thin` |

### Finding Icon Names

1. Browse icons at [phosphoricons.com](https://phosphoricons.com/)
2. Convert PascalCase to kebab-case: `CaretDown` → `caret-down`
3. Add the `ph:` prefix: `ph:caret-down`
4. Add variant suffix if needed: `ph:caret-down-fill`

## Client-Side: Direct Usage

For custom React components, use `@iconify/react` directly:

```tsx
import { Icon } from "@iconify/react";

function MyComponent() {
  return <Icon icon="ph:users" width={20} height={20} />;
}
```

## Why Not Direct React Imports?

Previous versions used direct React component imports from `@phosphor-icons/react`:

```typescript
// ❌ Old pattern — no longer recommended
import { FileTextIcon, UsersIcon } from "@phosphor-icons/react";
.meta({ icon: FileTextIcon })
```

The `c.icon("ph:name")` pattern is preferred because:

1. **Serializable** — Icons can be sent from server to client via JSON
2. **No bundling** — Iconify loads icons on demand, no tree-shaking needed
3. **Consistent** — Same pattern across all config surfaces (admin, dashboard, sidebar, actions)
4. **Lightweight** — Only loads icons that are actually rendered
