# Getting Started with Admin UI

This guide walks you through setting up the `@questpie/admin` package for your QUESTPIE CMS project.

## Prerequisites

You need a working QUESTPIE CMS backend. If you haven't set that up yet, see the [Backend Getting Started](/docs/getting-started) guide first.

## Installation

```bash
bun add @questpie/admin @questpie/tanstack-query @tanstack/react-query
```

## Project Structure

We recommend organizing your QUESTPIE configuration like this:

```
src/
└── questpie/
    ├── server/              # Backend CMS (q builder) — defines everything
    │   ├── cms.ts           # Main CMS instance with dashboard, sidebar
    │   ├── builder.ts       # Shared q builder instance
    │   ├── collections/     # Collection definitions (fields, admin, list, form, actions)
    │   └── globals/         # Global definitions
    │
    └── admin/               # Admin UI (qa builder) — renderers & branding only
        ├── admin.ts         # Branding, locale config
        └── builder.ts       # Shared builder instance
```

<Callout type="info">
Collections, dashboard, sidebar, and actions are all defined on the **server** using the `q` builder. The client-side `qa` builder only handles rendering components and branding.
</Callout>

## Step 1: Create the Server Builder

Create a shared builder instance for your server-side collections:

```typescript
// src/questpie/server/builder.ts
import { q } from "questpie";
import { adminModule } from "@questpie/admin/server";

export const qb = q({ name: "base" }).use(adminModule);
```

## Step 2: Define a Collection

Define your collection with fields, admin metadata, list view, and form layout — all on the server:

```typescript
// src/questpie/server/collections/posts.ts
import { qb } from "../builder";

export const posts = qb
  .collection("posts")
  .fields((f) => ({
    title: f.text({ required: true }),
    slug: f.text({ required: true }),
    content: f.richText({ localized: true }),
    status: f.select({
      options: [
        { label: "Draft", value: "draft" },
        { label: "Published", value: "published" },
      ],
      default: "draft",
    }),
    publishedAt: f.datetime(),
  }))
  .title(({ f }) => f.title)

  // Admin UI metadata
  .admin(({ c }) => ({
    label: { en: "Blog Posts" },
    icon: c.icon("ph:article"),
    group: "Content",
  }))

  // List/table view
  .list(({ v, f }) => v.table({
    columns: [f.title, f.status, f.publishedAt],
    defaultSort: { field: "publishedAt", direction: "desc" },
    searchable: ["title"],
    filterable: ["status"],
  }))

  // Create/edit form
  .form(({ v, f }) => v.form({
    sidebar: {
      position: "right",
      fields: [f.status, f.publishedAt],
    },
    sections: [
      {
        label: { en: "Content" },
        fields: [f.title, f.slug, f.content],
      },
    ],
  }));
```

## Step 3: Configure Dashboard & Sidebar

Wire everything together in your main CMS file:

```typescript
// src/questpie/server/cms.ts
import { q } from "questpie";
import { adminModule, adminRpc } from "@questpie/admin/server";
import { posts } from "./collections/posts";

export const cms = q({ name: "my-cms" })
  .use(adminModule)
  .collections({ posts })

  // Dashboard widgets
  .dashboard(({ d, c }) => d.dashboard({
    columns: 4,
    items: [
      d.stats({
        collection: "posts",
        label: { en: "Total Posts" },
        icon: c.icon("ph:article"),
        span: 1,
      }),
      d.stats({
        collection: "posts",
        label: { en: "Published" },
        icon: c.icon("ph:check-circle"),
        filter: { status: "published" },
        span: 1,
      }),
    ],
  }))

  // Sidebar navigation
  .sidebar(({ s, c }) => s.sidebar({
    sections: [
      s.section({
        id: "content",
        title: { en: "Content" },
        icon: c.icon("ph:files"),
        items: [
          { type: "collection", collection: "posts" },
        ],
      }),
    ],
  }))

  .auth({
    emailAndPassword: { enabled: true },
    baseURL: process.env.APP_URL,
    basePath: "/api/cms/auth",
    secret: process.env.BETTER_AUTH_SECRET,
  })
  .build({
    db: { url: process.env.DATABASE_URL },
    storage: { basePath: "/api/cms" },
  });

export type AppCMS = typeof cms;
```

## Step 4: Create the Client Admin Config

The client-side only configures branding and locale:

```typescript
// src/questpie/admin/admin.ts
import { qa, adminModule } from "@questpie/admin/client";
import type { AppCMS } from "@/questpie/server/cms";

export const admin = qa<AppCMS>()
  .use(adminModule)
  .branding({
    name: "My CMS",
  })
  .locale({
    default: "en",
    supported: ["en"],
  });

// Type registry for global type inference
declare module "@questpie/admin/client" {
  interface AdminTypeRegistry {
    cms: AppCMS;
    admin: typeof admin;
  }
}
```

## Step 5: Setup Styles

Import admin styles in your CSS:

```css
/* src/styles.css */
@import "tailwindcss";
@import "tw-animate-css";
@import "shadcn/tailwind.css";
@import "@questpie/admin/styles/index.css";

/* IMPORTANT: Tell Tailwind to scan admin package */
@source "../node_modules/@questpie/admin/dist";

/* Your theme variables... */
```

## Step 6: Mount in React

Create the admin route (example with TanStack Router):

```tsx
// routes/admin.tsx
import { AdminLayoutProvider } from "@questpie/admin/client";
import { Link, Outlet, useLocation } from "@tanstack/react-router";
import { admin } from "@/questpie/admin/admin";
import { cmsClient } from "@/lib/cms-client";
import { queryClient } from "@/lib/query-client";

export function AdminLayout() {
  const location = useLocation();

  return (
    <AdminLayoutProvider
      admin={admin}  // Pass the builder directly
      client={cmsClient}
      queryClient={queryClient}
      LinkComponent={Link}
      activeRoute={location.pathname}
      basePath="/admin"
    >
      <Outlet />
    </AdminLayoutProvider>
  );
}
```

## Step 7: Create the CMS Client

```typescript
// src/lib/cms-client.ts
import { createClient } from "questpie/client";
import type { AppCMS } from "@/questpie/server/cms";

export const cmsClient = createClient<AppCMS>({
  baseUrl: "/api/cms",
});
```

## Verifying the Setup

Start your development server and navigate to `/admin`. You should see:

1. The sidebar with your configured sections
2. Collection list pages with tables
3. Create/edit forms with your field configurations
4. Dashboard with your configured widgets
