# Email

QUESTPIE includes a mailer service for sending transactional emails. Define templates with a handler function that returns `{ subject, html }`, validate input with Zod, and send them via SMTP or log to the console during development.

## Why This Exists

Applications often need to send emails — appointment confirmations, password resets, notifications. QUESTPIE's email system lets you define type-safe templates and send them through `app.email`. Handlers receive full AppContext (db, collections, services) so you can fetch data directly inside the template.

## Setup

Configure the email adapter in `runtimeConfig()`:

```ts
// src/questpie/server/questpie.config.ts
import { runtimeConfig, SmtpAdapter, ConsoleAdapter } from "questpie";

export default runtimeConfig({
  db: { url: process.env.DATABASE_URL! },
  app: { url: process.env.APP_URL! },
  email: {
    adapter: process.env.NODE_ENV === "production"
      ? new SmtpAdapter({
          transport: {
            host: process.env.SMTP_HOST!,
            port: Number(process.env.SMTP_PORT || "587"),
            secure: true,
            auth: {
              user: process.env.SMTP_USER!,
              pass: process.env.SMTP_PASS!,
            },
          },
        })
      : new ConsoleAdapter({ logHtml: false }),
  },
});
```

## Defining Email Templates

Templates are defined with `email()` and a `handler` that returns `{ subject, html, text? }`:

```ts
// emails/appointment-confirmation.ts
import { email } from "questpie";
import { z } from "zod";

export default email({
  name: "appointment-confirmation",
  schema: z.object({
    customerName: z.string(),
    date: z.string(),
    time: z.string(),
    barberName: z.string(),
    serviceName: z.string(),
  }),
  handler: ({ input }) => ({
    subject: `Appointment confirmed for ${input.date}`,
    html: `
      <div>
        <h1>Hi ${input.customerName}!</h1>
        <p>Your appointment has been confirmed:</p>
        <ul>
          <li>Date: ${input.date} at ${input.time}</li>
          <li>Barber: ${input.barberName}</li>
          <li>Service: ${input.serviceName}</li>
        </ul>
      </div>
    `,
  }),
});
```

### Async Data Fetching

Handlers receive full AppContext, so you can query the database:

```ts
export default email({
  name: "order-receipt",
  schema: z.object({ orderId: z.string() }),
  handler: async ({ input, collections }) => {
    const order = await collections.orders.findOne({
      where: { id: input.orderId },
      with: { items: true },
    });

    return {
      subject: `Receipt for Order #${input.orderId.slice(0, 8)}`,
      html: `<h1>Order confirmed</h1><p>Total: $${order?.total}</p>`,
    };
  },
});
```

### Using React (Optional)

If you prefer React for complex layouts, install `@react-email/render` and call it inside the handler:

```ts
import { email } from "questpie";
import { render } from "@react-email/render";
import { z } from "zod";
import { WelcomeTemplate } from "./components/welcome";

export default email({
  name: "welcome",
  schema: z.object({ name: z.string() }),
  handler: async ({ input }) => ({
    subject: `Welcome, ${input.name}!`,
    html: await render(<WelcomeTemplate name={input.name} />),
  }),
});
```

Templates placed in the `emails/` directory are automatically discovered by codegen.

## Sending Emails

### Using Templates

```ts
// In a hook, function handler, or job
await email.sendTemplate({
  template: "appointmentConfirmation",
  to: "customer@example.com",
  input: {
    customerName: "Jane",
    date: "March 15, 2025",
    time: "10:00 AM",
    barberName: "John",
    serviceName: "Haircut",
  },
});
```

### Raw Email

```ts
await app.email.send({
  to: "user@example.com",
  subject: "Hello",
  html: "<h1>Hello World</h1>",
});
```

### Previewing Templates

Use `renderTemplate()` to get the rendered output without sending:

```ts
const result = await app.email.renderTemplate({
  template: "appointmentConfirmation",
  input: { customerName: "Jane", date: "March 15", ... },
});

console.log(result.subject); // "Appointment confirmed for March 15"
console.log(result.html);    // "<div>..."
```

## Adapters

### SmtpAdapter

Production email via NodeMailer:

```ts
new SmtpAdapter({
  transport: {
    host: "smtp.example.com",
    port: 587,
    secure: true,
    auth: { user: "...", pass: "..." },
  },
})
```

### ConsoleAdapter

Development-only — logs emails to the console:

```ts
new ConsoleAdapter({ logHtml: false })
```

### Ethereal Test Adapter

For testing with a real SMTP server that catches all emails:

```ts
import { createEtherealSmtpAdapter } from "questpie";

const adapter = await createEtherealSmtpAdapter();
// Logs preview URLs for each sent email
```

## Common Mistakes

- **No adapter configured** — Without an email adapter, `email.send()` will fail. Use `ConsoleAdapter` for development.
- **Missing template file** — Templates must be placed in the `emails/` directory and export a default `email()` definition to be discovered by codegen.
- **Sending emails synchronously in hooks** — Use background jobs for non-critical emails to avoid slowing down API responses.
- **Forgetting to return `subject`** — The handler must return both `subject` and `html`.
