# Localization

QUESTPIE supports both content localization and admin locale configuration.

## Content Locales

Content locales are configured via the `locale.ts` convention file:

```ts
// src/questpie/server/locale.ts
import { locale } from "#questpie";

export default locale({
  locales: [
    { code: "en", label: "English", fallback: true },
    { code: "sk", label: "Slovenčina" },
  ],
  defaultLocale: "en",
});
```

Use `localized: true` on fields that should be translated:

```ts
title: f.text({ label: { en: "Title" }, localized: true });
```

## Admin Locales

Admin UI locale is configured via the `admin-locale.ts` convention file:

```ts
// src/questpie/server/admin-locale.ts
import { adminLocale } from "#questpie";

export default adminLocale({
  locales: ["en", "sk"],
  defaultLocale: "en",
});
```

## Practical Split

- Content locale config belongs in `locale.ts` — defines which content languages are available
- Admin locale config belongs in `admin-locale.ts` — defines which UI languages the admin panel supports
