# Modules and Extensions

QUESTPIE is designed to be modular. Modules are pre-built packages that add collections, functions, jobs, admin features, and more to your app. Your own code follows a file convention that codegen discovers and assembles into a typed app.

## Core Concepts

### Modules

A module is a static object that bundles collections, functions, jobs, and configuration. Modules are registered in `modules.ts`:

```ts
// modules.ts
import { adminModule, auditModule } from "@questpie/admin/server";

export default [adminModule, auditModule] as const;
```

### File Convention

Your app's entities are organized by type in the server directory:

```text
src/questpie/server/
├── questpie.config.ts       # Runtime config: database, adapters, secrets
├── modules.ts               # Module dependencies
├── branding.ts              # Admin panel branding
├── sidebar.ts               # Admin sidebar configuration
├── dashboard.ts             # Admin dashboard widgets
├── locale.ts                # Content locale configuration
├── admin-locale.ts          # Admin UI locale
├── auth.ts                  # Authentication configuration
├── collections/             # One file per collection
│   ├── barbers.ts
│   ├── services.ts
│   └── appointments.ts
├── globals/                 # One file per global
│   └── site-settings.ts
├── functions/               # One file per function
│   ├── create-booking.ts
│   └── get-active-barbers.ts
├── jobs/                    # One file per background job
│   └── send-confirmation.ts
├── emails/                  # One file per email template
│   └── appointment-confirmation.ts
├── blocks/                  # One file per content block
│   └── services.ts
├── services/                # One file per custom service
│   └── blog.ts
└── .generated/              # Auto-generated by codegen
    ├── index.ts             # App entrypoint with types
    └── factories.ts         # Typed factories (collection, global, etc.)
```

### Codegen

Run `questpie generate` (or `questpie dev` for watch mode) to scan the file convention and produce the `.generated/` directory. This generates:

- **App entrypoint** — imports all entities, creates the app, exports types
- **AppContext augmentation** — types all handler contexts automatically
- **Typed factories** — `collection()`, `global()`, `block()` etc. from `#questpie`
- **Registry types** — `AppCollections`, `AppGlobals`, `AppJobs`, etc.

## Runtime Config

The `questpie.config.ts` file handles runtime-only concerns:

```ts
// questpie.config.ts
import { runtimeConfig, pgBossAdapter, SmtpAdapter } from "questpie";

export default runtimeConfig({
  app: { url: process.env.APP_URL || "http://localhost:3000" },
  db: { url: process.env.DATABASE_URL! },
  secret: process.env.BETTER_AUTH_SECRET!,
  queue: { adapter: pgBossAdapter({ connectionString: process.env.DATABASE_URL! }) },
  email: {
    adapter: new SmtpAdapter({
      transport: { host: "localhost", port: 1025, secure: false },
    }),
  },
  cli: {
    migrations: { directory: "./src/migrations" },
    seeds: { directory: "./src/seeds" },
  },
});
```

Entity definitions (collections, globals, etc.) live in their own files — not in the config.

## Built-in Modules

### `starterModule`

Included by default. Provides core functionality:

- `user` collection with auth integration
- `assets` collection for file uploads
- Default access control (require authentication)
- Auth session management

### `adminModule`

The admin panel module. Adds:

- Admin UI server functions (config, setup, preview, etc.)
- Admin-specific field types (`upload`, `richText`, `blocks`)
- Admin preferences and saved views collections
- Default sidebar, dashboard, and branding support

### `auditModule`

Audit logging module. Adds:

- `auditLog` collection
- Automatic change tracking hooks
- Audit cleanup job
- Dashboard widget for recent activity

## Module Registration

```ts
// modules.ts
import { adminModule, auditModule } from "@questpie/admin/server";

export default [adminModule, auditModule] as const;
```

Modules are static objects — no factory function calls needed. Their collections, functions, and jobs are merged with your app's entities during codegen.

## File Convention Details

### Convention Files

These single files are discovered by name:

| File | Purpose | Import |
|------|---------|--------|
| `questpie.config.ts` | Runtime config | `runtimeConfig()` from `"questpie"` |
| `modules.ts` | Module dependencies | Static module imports |
| `branding.ts` | Admin branding | `branding()` from `"#questpie"` |
| `sidebar.ts` | Admin sidebar | `sidebar()` from `"#questpie"` |
| `dashboard.ts` | Admin dashboard | `dashboard()` from `"#questpie"` |
| `locale.ts` | Content locales | `locale()` from `"#questpie"` |
| `admin-locale.ts` | Admin UI locale | `adminLocale()` from `"#questpie"` |
| `auth.ts` | Auth configuration | Auth setup |

### Entity Directories

These directories contain one entity per file:

| Directory | Entity | Factory |
|-----------|--------|---------|
| `collections/` | Collections | `collection()` from `"#questpie"` |
| `globals/` | Globals | `global()` from `"#questpie"` |
| `functions/` | Server functions | `fn()` from `"questpie"` |
| `jobs/` | Background jobs | `job()` from `"questpie"` |
| `emails/` | Email templates | `email()` from `"questpie"` |
| `blocks/` | Content blocks | `block()` from `"#questpie"` |
| `services/` | Custom services | `defineService()` from `"questpie"` |

### Import Convention

- **Typed factories** (`collection`, `global`, `block`, `sidebar`, etc.) come from `"#questpie"` — these are generated with your app's specific types.
- **Framework utilities** (`fn`, `job`, `email`, `defineService`, `runtimeConfig`) come from `"questpie"` — these are generic framework functions.

## Creating Custom Modules

For package authors, modules are objects conforming to the module interface:

```ts
import type { QuestpieModule } from "questpie";

export const myModule: QuestpieModule = {
  name: "my-module",
  collections: {
    myCollection: myCollectionDefinition,
  },
  functions: {
    myFunction: myFunctionDefinition,
  },
  jobs: {
    myJob: myJobDefinition,
  },
};
```

Module entities are merged with user-defined entities. User definitions take precedence when names conflict.

## Common Mistakes

- **Using factory calls for modules** — Modules are static objects (`adminModule`), not factory functions (`admin()`). Don't call them.
- **Putting entity definitions in config** — Collections, globals, functions go in their own files under the file convention directories. `questpie.config.ts` is for runtime config only.
- **Forgetting to run codegen** — After adding new files, run `questpie generate` to update the generated types and app entrypoint.
- **Importing from wrong source** — Use `"#questpie"` for typed factories and `"questpie"` for framework utilities.
