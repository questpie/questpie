# Getting Started

Get up and running with QUESTPIE CMS in minutes. This guide will walk you through installation, creating your first collection, and running the server.

## Requirements

| Requirement    | Version | Notes                                          |
| -------------- | ------- | ---------------------------------------------- |
| **Bun**        | 1.0+    | Recommended runtime ([bun.sh](https://bun.sh)) |
| **Node.js**    | 20+     | Alternative to Bun                             |
| **PostgreSQL** | 15+     | Required for database                          |
| **TypeScript** | 5.0+    | For type safety                                |

## Installation

### 1. Create a New Project

```bash
mkdir my-cms && cd my-cms
bun init -y
```

### 2. Install Dependencies

```bash
# Core CMS + Drizzle ORM
bun add @questpie/cms drizzle-orm@beta zod

# Development tools
bun add -D drizzle-kit typescript @types/bun

# Choose your framework adapter
bun add @questpie/hono hono        # Hono
# OR
bun add @questpie/elysia elysia    # Elysia
# OR
bun add @questpie/next next        # Next.js
```

### 3. Setup TypeScript

```json
// tsconfig.json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "outDir": "dist",
    "declaration": true
  },
  "include": ["src/**/*", "cms.config.ts"]
}
```

### 4. Setup Environment

```bash
# .env
DATABASE_URL="postgresql://user:password@localhost:5432/mydb"
```

## Quick Start: Your First Collection

### 1. Define a Collection

```typescript
// src/collections/posts.ts
import { defineCollection } from "@questpie/cms/server";
import { varchar, text, boolean, timestamp } from "drizzle-orm/pg-core";

export const posts = defineCollection("posts")
  .fields({
    title: varchar("title", { length: 255 }).notNull(),
    slug: varchar("slug", { length: 100 }).notNull().unique(),
    content: text("content"),
    isPublished: boolean("is_published").default(false).notNull(),
    publishedAt: timestamp("published_at", { mode: "date" }),
  })
  .title((row) => row.title); // Display title in admin
```

### 2. Create CMS Instance

```typescript
// src/cms.ts
import { defineQCMS } from "@questpie/cms/server";
import { posts } from "./collections/posts";

export const cms = defineQCMS({ name: "my-cms" })
  .collections({ posts })
  .build({
    db: {
      url: process.env.DATABASE_URL!,
    },
  });

// Export type for client
export type CMS = typeof cms;
```

### 3. Create Config File

```typescript
// cms.config.ts
import { cms } from "./src/cms";
export default { qcms: cms };
```

### 4. Create Server (Hono Example)

```typescript
// src/server.ts
import { Hono } from "hono";
import { questpieHono } from "@questpie/hono";
import { cms } from "./cms";

const app = new Hono();

// Mount CMS routes at /cms
app.route("/cms", questpieHono(cms));

// Health check
app.get("/health", (c) => c.json({ status: "ok" }));

export default {
  port: 3000,
  fetch: app.fetch,
};
```

### 5. Add Scripts

```json
// package.json
{
  "scripts": {
    "dev": "bun --watch src/server.ts",
    "qcms": "bun --bun ./node_modules/@questpie/cms/dist/cli/index.js"
  }
}
```

### 6. Generate & Run Migrations

```bash
# Generate migration from your schema
bun qcms migrate:generate

# Run the migration
bun qcms migrate:up
```

### 7. Start the Server

```bash
bun dev
```

Your CMS is now running at `http://localhost:3000/cms`.

## Test the API

```bash
# Create a post
curl -X POST http://localhost:3000/cms/collections/posts \
  -H "Content-Type: application/json" \
  -d '{"title": "Hello World", "slug": "hello-world", "content": "My first post!"}'

# List posts
curl http://localhost:3000/cms/collections/posts

# Get single post
curl http://localhost:3000/cms/collections/posts/{id}
```

## Project Structure

Here's the recommended structure for a QUESTPIE CMS project:

```
my-cms/
├── src/
│   ├── collections/       # Collection definitions
│   │   ├── index.ts       # Export all collections
│   │   └── posts.ts
│   ├── globals/           # Global settings
│   │   └── site-settings.ts
│   ├── jobs/              # Background jobs
│   │   └── send-email.ts
│   ├── cms.ts             # CMS configuration
│   └── server.ts          # HTTP server
├── migrations/            # Generated migrations
├── cms.config.ts          # CLI config
├── .env                   # Environment variables
├── package.json
└── tsconfig.json
```

## Adding Authentication

Enable authentication with Better Auth:

```typescript
// src/cms.ts
import { defineQCMS } from "@questpie/cms/server";
import { posts } from "./collections/posts";

export const cms = defineQCMS({ name: "my-cms" })
  .collections({ posts })
  .auth({
    emailAndPassword: {
      enabled: true,
    },
    // Optional: add OAuth providers
    // socialProviders: {
    //   google: {
    //     clientId: process.env.GOOGLE_CLIENT_ID!,
    //     clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    //   },
    // },
  })
  .build({
    db: {
      url: process.env.DATABASE_URL!,
    },
  });
```

Now protected endpoints require authentication. See [Authentication Guide](/docs/guides/authentication) for details.

## Adding Background Jobs

Enable the queue for background processing:

```typescript
// src/cms.ts
import { defineQCMS, defineJob, pgBossAdapter } from "@questpie/cms/server";

const sendWelcomeEmail = defineJob({
  name: "send-welcome-email",
  schema: z.object({ userId: z.string() }),
  handler: async ({ data }) => {
    // Send email logic
    console.log(`Sending welcome email to user ${data.userId}`);
  },
});

export const cms = defineQCMS({ name: "my-cms" })
  .collections({ posts })
  .jobs({ sendWelcomeEmail })
  .build({
    db: { url: process.env.DATABASE_URL! },
    queue: {
      adapter: pgBossAdapter({
        connectionString: process.env.DATABASE_URL!,
      }),
    },
  });
```

See [Background Jobs Guide](/docs/guides/queue-jobs) for more patterns.
